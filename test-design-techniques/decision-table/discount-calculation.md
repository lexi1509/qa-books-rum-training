### Оплата наличными
- Дополнительная скидка: **2%**
- Суммируется со скидкой по карте лояльности

### Ограничения
- К одному заказу может быть применена только одна карта лояльности

## Найденные баги

### Баг 1: Неверная скидка для без карты + оплата картой при сумме 10000
- **Ожидаемая скидка:** 20%
- **Фактическая скидка:** 15%
- **Ожидаемая сумма:** 8000
- **Фактическая сумма:** 8500

### Баг 2: Не учитывается скидка при оплате наличными для без карты при сумме 10000
- **Ожидаемая скидка:** 22%
- **Фактическая скидка:** 17%

### Баг 3: Неверная скидка для стандартной карты + оплата картой при сумме 10000
- **Ожидаемая скидка:** 30%
- **Фактическая скидка:** 20%

### Баг 4: Неверная скидка для золотой карты + оплата картой при сумме 10000
- **Ожидаемая скидка:** 35%
- **Фактическая скидка:** 25%

### Баг 5: Неверный расчёт для золотой карты + наличные при сумме 4999
- **Ожидаемая сумма:** 4399.12
- **Фактическая сумма:** 4149.17

### Баг 6: Неверная скидка для золотой карты + наличные при сумме 10000
- **Ожидаемая скидка:** 37%
- **Фактическая скидка:** 27%

## Выводы и рекомендации

1. **Частые баги на граничных значениях.** Особенно при переходе через сумму 5000 и 10000.
2. **Проблемы с суммированием скидки при оплате наличными.** Часто не учитывается дополнительная скидка 2%.
3. **Рекомендация:** В требованиях чётко прописать диапазоны сумм заказа (например, до 4999 включительно, от 5000 включительно), чтобы исключить неверные расчёты скидок при попадании суммы на границы.
